# HELP — какой скилл на каком этапе

Быстрая шпаргалка. Подробности — в `START-HERE.md`.

## Поток проекта (слева направо)

```
идея → /kickoff → /research → /grill-me → /to-issues → /triage → /implement → /ship → /sync
```

## Этапы

| Этап | Команда | Что делает |
|---|---|---|
| Захват идеи (телефон) | «сохрани в inbox» | кладёт идею issue-задачей с лейблом `inbox` |
| Раскрутка идеи (комп) | `/kickoff` | inbox/новая идея → репо + план + слайсы на доске |
| Быстрая задача | `/route` | задача в правильный репо с лейблами (без раскрутки) |
| Разведка | `/research` | фоновый ресёрч по первоисточникам → md с цитатами |
| Прожарить замысел | `/grill-me`, `/grilling` | стресс-тест плана вопросами |
| Прототип | `/prototype` | throwaway-код, отвечающий на один вопрос |
| План → задачи | `/to-issues` | tracer-bullet слайсы на доске |
| Разбор задач | `/triage` | пометить ready-for-agent, написать бриф |
| Реализация | `/implement` | взять ready-задачу → код → ревью → коммит |
| Отладка | `/diagnosing-bugs` | feedback-loop сначала, потом гипотезы |
| Релиз | `/ship` | release notes → tag → push → до «живо» |
| Конец сессии | `/sync` | work-records в issues, статусы, push штаба |
| Передать сессию | `/handoff` | сводка для следующего агента (в issue) |

## Роли (надеть шляпу)

У каждой роли есть одноимённая команда `/<role>` — сразу надевает роль на сессию
(роли задаются при `/hub-init`, список — в HQ.md).
`/dept <role>` — сменить роль проекта, `/dept --new <role>` — завести новую роль.

## Система (реже)

| Команда | Что делает |
|---|---|
| `/project-register` | подключить репо (`--scan` — все сразу) |
| `/hq-doctor` | проверить/починить инварианты (симлинки, реестр, лейблы) |
| `/weekly` | `--plan` / `--retro` — недельные ритуалы |
| `/learn` | правило из урока → в нужный слой (проект/роль/штаб) |
| `/hub-init` | первичная настройка штаба (разово) |
