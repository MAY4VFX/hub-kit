# Стандарт отдела

Регламент: как устроен отдел (роль), как создать новый, какие файлы в нём возможны.
Отделы создают ТОЛЬКО скиллы `/hub-init` (при онбординге) и `/dept --new <роль>` —
оба следуют этому стандарту. Агент не выдумывает свою структуру.

## Структура

```
departments/<slug>/          # slug: латиница, kebab-case
├── role.md                  # ОБЯЗАТЕЛЬНЫЙ. Единственный файл-слой: на него смотрит
│                            # симлинк .dept.md проектов, он попадает в контекст агента
└── <материалы>.md           # опциональные, по белому списку ниже
```

## Белый список материалов (создавать по мере надобности, не заранее)

| Файл | Что это | Кто обновляет |
|---|---|---|
| `inventory.md` | Карта ресурсов отдела (хосты, сервисы, аккаунты) | только профильный скилл-скан (напр. `/infra-scan`) |
| `playbooks.md` | Пошаговые процедуры «как мы делаем X» | явная задача или `/learn`, если урок = процедура |
| `decisions.md` | Журнал решений отдела (дата, решение, почему) | явная задача |
| `contacts.md` | Внешние сервисы/люди/аккаунты отдела (БЕЗ секретов) | явная задача |

Нужен файл вне списка — сначала добавь его сюда (в стандарт) с владельцем, потом создавай.

## Правила

1. **В контекст автоматически попадает только `role.md`** — держи его коротким
   (< 40 строк). Всё тяжёлое — в соседние файлы, из role.md — ссылка одной строкой
   («карта инфраструктуры — inventory.md»); агент читает их по ссылке, когда нужно.
2. **role.md не перечисляет проекты** — проекты роли смотри в HQ.md (иначе устареет).
3. **Секретов в папке отдела нет никогда** — только имена переменных.
4. У каждого материала есть владелец (скилл или «явная задача») — файлы без владельца гниют.

## Секции role.md (шаблон `_role.md.tmpl`)

```markdown
# Роль: <имя>            ← identity: «Ты — агент роли X. Штаб: <hub>»
## Зона ответственности   ← 1–3 строки + указатель на HQ.md
## Правила роли           ← пополняется /learn; императив, с «почему»
## Задачи                 ← лейбл dept:<slug>, где смотреть открытые
## Материалы отдела       ← ссылки на файлы папки (если есть)
## Рекомендуемые скиллы   ← чем эта роль обычно работает
```

## Чек-лист создания нового отдела (`/dept --new` делает сам)

1. Спросить у владельца: имя, slug, домен одной строкой.
2. `departments/<slug>/role.md` из `_role.md.tmpl` (примеры — `examples/` в hub-kit).
3. Лейбл `dept:<slug>` в hub-репо.
4. Коммит + push штаба. Материалы — НЕ создавать заранее (пустые файлы = мусор).
