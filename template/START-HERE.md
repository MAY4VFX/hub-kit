# START HERE — как жить с этой системой

Это твой штаб. Он один на все проекты, на все компьютеры и на телефон.
Всё, что здесь описано, делается словами в Claude Code — команды учить не обязательно,
достаточно понять картинку.

## Картинка

- Каждый проект — отдельный репозиторий на GitHub, локально все лежат рядом в одной
  папке (`{{REPOS_ROOT}}`).
- У тебя есть **роли** (см. `departments/`) — «шляпы», которые агент надевает, работая
  над проектом: {{ROLES_LIST}}. Проект в любой момент носит одну активную роль, и её
  можно менять — проект не переезжает, меняется только контекст агента.
- **Память системы — GitHub**: задачи = issues, общая доска = GitHub Project. Агент в
  начале сессии читает доску («что было»), в конце — записывает туда прогресс. Поэтому
  можно бросить работу на одном компьютере и продолжить на другом или с телефона.

## Три главные команды

1. **`/kickoff`** — «у меня идея». Превращает идею в проект: создаёт репозиторий,
   сохраняет твой замысел как основу плана, раскладывает работу на задачи на доске.
   Работает и с идеями, накиданными с телефона (см. ниже).
2. **`/route`** — «надо не забыть сделать X». Кладёт задачу в правильный проект с
   правильными метками. Никогда ничего не потеряется.
3. **`/sync`** — «я закончил на сегодня». Записывает, что сделано, в задачи и доску.
   Это самое важное: без него следующая сессия не узнает, что было.

## Идея с телефона

Разгоняешь идею с любым агентом на телефоне (claude.ai / Claude Code web) и в конце
говоришь: **«сохрани в inbox»** — агент положит её issue-задачей в штаб. Когда сядешь
за компьютер, скажи `/kickoff` — он покажет накопившиеся идеи и раскрутит выбранную.

## Реже, но полезно

- `/project-register` — подключить существующий репозиторий в систему
  (`/project-register --scan` — найти и подключить все сразу).
- `/dept <роль>` — сменить активную роль проекта (с передачей открытых задач).
- `/dept --new <роль>` — завести новую роль.
- `/hq-doctor` — если кажется, что что-то разъехалось: проверит и починит.

## Что где лежит

| Файл | Что это |
|---|---|
| `CLAUDE.md` | Правила для агентов + машинный конфиг (`## Hub Config`) |
| `HQ.md` | Реестр всех проектов: роль, репо, путь, домен |
| `departments/*.md` | Слои ролей — правила каждой «шляпы» |

Реестр и роли руками лучше не править — это делают команды выше, чтобы ничего не
разъехалось. Если очень надо — просто скажи агенту, что поменять.
