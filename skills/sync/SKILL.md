# Sync — память в GitHub

GitHub issues — источник правды о задачах; Project-доска — о том, что активно.
Без write-back следующая сессия (на любой машине) не узнает, что было.

## Режим write (конец сессии / «зафиксируй»)

1. **Собери**: что сделано в этой сессии (коммиты `git log --oneline @{u}.. || -10`,
   изменённые репо, решения).
2. **Найди issues**: открытые задачи затронутых репо (`gh issue list -s open`) +
   доска. Сопоставь работу с задачами.
3. **План синка** — покажи пользователю таблицей: `issue | что запишу | новый статус`.
   По Hub Config `authorization: ask-each-time` — жди подтверждения.
4. **Исполни**:
   - **work-record** комментарием в каждый issue с реальной работой: что сделано +
     ссылки на коммиты `<repo>@<sha>`. Механические перестановки лейблов — без
     комментария;
   - статусы на доске (`gh project item-edit`);
   - работа без подходящего issue → предложи создать через `/route`;
   - инварианты не рушить: не снимать epic, не менять чужие лейблы.
5. **Штаб**: dirty → коммит + push. Затронутые репо не запушены → скажи явно.

## Режим read («что по X?»)

1. Поиск по репо из HQ.md: `gh search issues "<X>" --owner <owner> --state all` +
   доска.
2. Ответ: `<repo>#N «title»` · статус · последняя активность · гист последнего
   work-record. Дрейф (issue открыт, но код давно ушёл вперёд) — отметь.

## Red flags

| Собираешься… | Вместо этого… |
|---|---|
| Закрыть issue без work-record | Сначала комментарий с итогом и коммитами |
| Записать прогресс в тело issue | Тело = постановка; прогресс = комментарии |
| Синкнуть без показа плана | ask-each-time: сначала план, потом запись |
| Сослаться на незапушенный коммит | Сначала push, потом ссылка |
