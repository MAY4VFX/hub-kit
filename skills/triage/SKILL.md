# Triage — стейт-машина задач

Готовит задачи к работе: человек триажит с телефона/на бегу, агент потом забирает
`ready-for-agent` без вопросов. Каждый комментарий триажа начинается с
`> *Сгенерировано ИИ при триаже.*`

## Роли (лейблы; создай отсутствующие в целевом репо)

Категория (ровно одна): `bug` | `enhancement`.
Состояние (ровно одно): `needs-triage` → `needs-info` | `ready-for-agent` |
`ready-for-human` | `wontfix`. `needs-info` возвращается в `needs-triage` после ответа.
Плюс инварианты хаба: epic-родитель, `dept:*`, доска (см. `/route`).

## Вызов

Естественным языком: «покажи что требует внимания», «разбери <repo>#42»,
«что готово для агентов?». Интерпретируй и действуй.

**Показать внимание**: три корзины, старые сверху — (1) без лейблов триажа,
(2) `needs-triage`, (3) `needs-info` с ответом репортера.

## Перевод в ready-for-agent — только с брифом

Бриф — секция `## Agent brief` в теле issue:
- **Контекст**: что это и почему сейчас; ссылки на код (`path:line`), решения, дампы.
- **Definition of done**: проверяемые пункты (тест проходит, команда выводит X).
- **Границы**: что НЕ трогать; известные грабли.
- **Проверка**: как агент сам убедится, что готово (команда/тест).
Бриф должен быть **долговечным**: агент без этой сессии в голове должен справиться.
Недостаточно данных для брифа → `needs-info` с конкретными вопросами репортеру.

## Red flags

| Собираешься… | Вместо этого… |
|---|---|
| ready-for-agent без брифа | Бриф обязателен — это контракт |
| Закрыть старый issue как мусор | `wontfix` предлагает человек, не ты |
| Два состояния на issue | Ровно одно; конфликт — спроси |
| Триажить, забыв инварианты хаба | epic + dept:* + доска (через /route-правила) |
