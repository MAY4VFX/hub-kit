# Learn — урок → нужный слой

Система самообучается, только если уроки записываются, и не гниёт, только если каждый
урок лежит в правильном слое. Слои (от узкого к широкому):

| Слой | Файл | Когда сюда |
|---|---|---|
| Проект | `<repo>/AGENTS.md` ниже identity-блока | Правда только про этот репо (его стек, деплой, конвенции) |
| Роль | `<штаб>/departments/<role>.md` | Правда для всех проектов роли (стиль работы «шляпы») |
| Штаб | `<штаб>/AGENTS.md` (Правила) | Правда для всей системы (ритуалы, git, issues) |
| Раннер | `~/.claude/CLAUDE.md` и аналоги | Правда про машину/пользователя вне системы хаба |

## Процесс

1. **Сформулируй правило**: 1–3 строки, императив, с «почему», если неочевидно.
   Из «ты опять забыл прогнать миграции» → «После правок в models/ прогони alembic
   upgrade head локально до коммита».
2. **Выбери слой** по таблице. Тест: «для кого это правда?» Один репо → проект; все
   проекты этой шляпы → роль; вся система → штаб. Сомневаешься между двумя — покажи
   оба варианта и спроси (узкий слой по умолчанию: правило легче поднять, чем сузить).
3. **Проверь дубли**: возможно, правило уже есть и его надо уточнить, а не добавить.
4. **Запиши**: в существующую подходящую секцию слоя, стилем соседних правил. Слой
   роли/штаба → коммит + push штаба (`docs(<слой>): <правило кратко>`).
5. Подтверди пользователю: что записано, куда, почему именно туда.

## Red flags

| Собираешься… | Вместо этого… |
|---|---|
| Всё валить в штаб «чтобы точно сработало» | Штаб растёт → все сессии платят токенами. Узкий слой |
| Записать «не делай так» без «как надо» | Правило = как надо (+ почему) |
| Продублировать существующее правило | Шаг 3: уточни старое |
| Записать разовое пожелание сессии | Разовое живёт в сессии, не в слоях |
