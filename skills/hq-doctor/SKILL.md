# HQ Doctor — инварианты системы

Read-only диагностика → отчёт → починка только с подтверждения.

## Проверки

По каждому репо из HQ.md (кроме самого штаба):

| # | Инвариант | Проверка |
|---|---|---|
| 1 | Репо склонирован в корне | `test -d <repos_root>/<name>/.git` |
| 2 | Симлинк есть, относительный, резолвится | `readlink .dept.md` + `test -e` |
| 3 | Симлинк указывает на роль из HQ.md | target == `departments/<role из HQ>/role.md` |
| 4 | Identity-блок в AGENTS.md (или CLAUDE.md) | grep `hub-kit identity block` |
| 5 | Лейблы `dept:<role>`, `epic` существуют | `gh label list -R` |
| 6 | Открытые issues репо имеют epic-родителя и dept-лейбл | `gh issue list` выборочно |
| 7 | Активные issues есть на доске | `gh project item-list` |

Плюс сам штаб: Hub Config парсится; `project_number` заполнен; роли из HQ.md существуют
в `departments/`; штаб не отстаёт от origin (`git fetch && git status`).
Плюс стандарт отделов (`departments/README.md`): в каждой папке отдела есть `role.md`;
файлы вне белого списка стандарта — флагай (не удаляй).

## Отчёт

Таблица `репо | проблема | как чиню`. Разделяй: **чиню сам** (битый симлинк, отсутствующий
лейбл, не на доске) vs **нужно решение человека** (репо из HQ.md нет на GitHub, роль
удалена, конфликт identity). Чинить — только после «да»; каждое исправление — отдельной
строкой в итоге. Изменения штаба — коммит + push.

## Red flags

- Не «чини» удалением строк из HQ.md — исчезнувший репо выноси человеку.
- Не создавай роли молча — отсутствующая роль это решение, не баг.
