# Kickoff — идея → проект

Одна воронка для обеих точек входа. Контейнер идеи — issue с лейблом `inbox` в hub-репо.

## Точки входа

- **`/kickoff`** (без аргументов): `gh issue list -R <hub_repo> -l inbox -s open` →
  покажи списком (номер, заголовок, гист, возраст) → пользователь выбирает.
- **`/kickoff <идея текстом>`**: ПЕРВЫМ шагом создай inbox-issue из идеи (запись в море
  до начала работы — сессия может умереть, идея нет). Затем обрабатывай его же.

## Пайплайн (для выбранного inbox-issue)

1. **Прочитай дамп целиком** (тело + комментарии + вложения).
2. **Короткий grill** (3–5 вопросов максимум, только по реальным дырам): цель, масштаб
   (задача или проект?), первый демонстрируемый результат, роль.
3. **Развилка**:
   - **Задача в существующий репо** → передай в `/route` (сошлись в issue на inbox-дамп),
     inbox-issue закрой ссылкой. Конец.
   - **Новый проект** → дальше.
4. **Репо**: `/project-register --new <name>` (создаст private-репо в корне, identity,
   симлинк выбранной роли, строку HQ.md, лейблы).
5. **Идея как основа плана**: сохрани дамп в `<repo>/docs/idea.md` (очисти от шума, но
   сохрани все решения и ссылки; источник: `<hub>#N`). Коммит — это «первые коммиты»
   проекта вместе с README/identity.
6. **Epic + слайсы**: создай epic-issue проекта; разбей первый этап на tracer-bullet
   слайсы (каждый — тонкий, но сквозной и демонстрируемый); issues по инвариантам
   `/route` (epic, dept-лейбл, доска). Покажи разбиение, спроси про гранулярность.
7. **Закрой inbox-issue**: комментарий «раскручено → <repo> epic #N» + закрыть.
8. **Финал**: скажи, что дальше — `cd <repo>` и работать по слайсам; `/sync` в конце.

## Red flags

| Собираешься… | Вместо этого… |
|---|---|
| Начать раскрутку идеи с компа без записи | Сначала inbox-issue, потом работа |
| Создать репо для «идеи», которая на самом деле задача | Развилка шага 3: route |
| Потерять сырой дамп при «очистке» | docs/idea.md хранит все решения и ссылки |
| Слайсы-«слои» (сначала вся схема, потом весь API…) | Tracer bullets: сквозные тонкие |
| Грилить 20 вопросами | 3–5, только реальные дыры |
