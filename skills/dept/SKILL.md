# Dept — надеть/сменить роль (шляпу)

Шляпа = контекст роли, а НЕ привязка к папке. Два способа её носить:
- **персистентно** (шляпа проекта): симлинк `.dept.md` в репо проекта → роль загружается
  сама при старте сессии в этом репо. Это дефолт для продуктовых репо.
- **эфемерно** (шляпа сессии): прочитать слой роли в текущий контекст — работает в ЛЮБОЙ
  папке (штаб, чужой репо, вообще без репо). Для ничейных/кросс-проектных задач роли,
  у которых нет своего продуктового репо (типичный кейс — infra: развернуть общий сервис).

Определи режим: запущен в репо с `.dept.md` и без флага → работа с симлинком (ниже);
запущен в штабе / без `.dept.md` / с флагом `--here` → эфемерная шляпа.

## Конфиг

Штаб и Hub Config — как в project-register. Текущая роль проекта: `readlink .dept.md`.
Целевая роль должна существовать в `departments/<slug>/` (кроме `--new`).

## /dept <role> --here — эфемерная шляпа (без репо, без симлинка)

Надеть роль на ТЕКУЩУЮ сессию, ничего не коммитя и не привязывая к папке.

1. Прочитай `<штаб>/departments/<role>/role.md` целиком в контекст.
2. Прочитай материалы роли, на которые ссылается role.md и которые нужны задаче
   (напр. `skills.md` — каким скиллом что делать, `map.md` — карта ресурсов).
3. С этого момента и до конца сессии ты — агент этой роли: следуй её правилам,
   используй её скиллы. Ничего не перелинковывается, штаб не мутируется.
4. Задачу бери из трекера по её ссылке (`<repo>#N`, лейбл `dept:<role>`) независимо от
   того, в какой ты папке. Ничейные инфра-задачи живут в трекере штаба под infra-epic.

Это «session-слой» — сильнейший и неперсистентный: снимается закрытием сессии.

## /dept <role> — полный handoff

1. **Старая роль**: `readlink .dept.md` → `<old>`. Если совпадает с целевой — скажи и выйди.
2. **Write-back**: найди открытые issues `dept:<old>` этого репо
   (`gh issue list -l dept:<old> -s open`). По каждому, где есть незафиксированная
   работа — work-record-комментарий «передано в <new>: <статус одной строкой>».
   Спроси, какие закрыть.
3. **Перелинковка**: `ln -sfn ../<штаб-имя>/departments/<new>/role.md .dept.md` +
   коммит в проект: `chore: dept <old> → <new>`.
4. **Штаб**: обнови колонку «роль» строки проекта в HQ.md; коммит + push штаба
   (другие машины узнают о смене фазы).
5. **Задача новой роли**: предложи создать/найти issue `dept:<new>` со ссылкой
   `needs <repo>#<N>` на переданные.
6. **Подгрузи контекст СЕЙЧАС**: прочитай `departments/<new>/role.md` целиком — контекст-файлы
   раннер грузит только на старте сессии, поэтому с этого момента работай по
   правилам новой роли без рестарта.

## /dept <role> --quick

Только шаги 3–4 (примерить шляпу без бюрократии) + шаг 6.

## /dept --new <role>

Структура отдела СТАНДАРТИЗОВАНА — следуй `departments/README.md` штаба (стандарт:
папка + обязательный role.md, материалы только из белого списка, не заранее).

1. Спроси домен роли одной строкой.
2. Создай `departments/<slug>/role.md` из `${CLAUDE_PLUGIN_ROOT}/template/departments/_role.md.tmpl`
   (покажи подходящий пример из `examples/`, если есть).
3. **Одноимённый hat-скилл**: `skills/<slug>/SKILL.md` из
   `${CLAUDE_PLUGIN_ROOT}/template/departments/_dept-hat.md.tmpl` (подставь `<slug>`).
   Теперь `/<slug>` сразу надевает эту роль.
4. Лейбл `dept:<slug>` в hub-репо (в репо проектов — добавит project-register/route
   по мере надобности).
5. Коммит + push штаба. **Скажи пользователю**: чтобы новый скилл `/<slug>` появился —
   обновить плагин (`claude plugin update <hub>@<hub>`) и начать новую сессию.
6. Если запущен в репо проекта — предложи сразу `/dept <slug>`.

## /dept (без аргументов)

Покажи: текущую роль (проекта или эфемерную), список доступных ролей с доменами,
открытые issues текущей роли.

## Red flags — STOP

| Собираешься… | Вместо этого… |
|---|---|
| Завести home-репо ради роли без своего кода | Не надо: `/dept <role> --here` надевает шляпу в штабе |
| Сменить роль проекта, бросив открытые issues старой без записи | Шаг 2 обязателен (кроме --quick/--here) |
| Создать симлинк на несуществующую роль | Сначала /dept --new |
| Абсолютный путь в симлинке | Только относительный |
| Продолжить сессию по правилам старой роли | Прочитать новый слой сразу |
