# Ship — релиз проекта

Запускается из репо проекта. Доводит работу до «выкачено и живо».

## Процесс

1. **Что выпускаем**: диапазон с прошлого тега (`git describe --tags --abbrev=0`;
   тегов нет — предложи схему `vX.Y.Z` и первый тег). Покажи `git log <last>..HEAD
   --oneline`.
2. **Чистота**: рабочее дерево чистое; тесты/линт зелёные (есть — прогони); закрытые
   этим релизом issues — найди по коммитам.
3. **Release notes**: сгруппируй по `feat / fix / breaking / прочее`, человеческим
   языком, со ссылками `<repo>#N`. Покажи черновик — правки?
4. **Релиз**: `git tag -a vX.Y.Z` → `git push --follow-tags` (авто-деплой с main —
   предупреди, что push = выкат) → `gh release create vX.Y.Z --notes "..."`.
5. **Контроль деплоя** — как выкатывается ЭТОТ проект, знает его AGENTS.md или слой
   роли (напр. роль infra знает Dokploy): найди инструкцию там и проследи до
   «живо» (статус деплоя, health-check, смоук). Не нашёл инструкции — спроси, как
   проверить, и предложи записать ответ в AGENTS.md проекта.
6. **Work-record**: в закрытые релизом issues — «выпущено в vX.Y.Z»; доска — Done.

## Red flags

| Собираешься… | Вместо этого… |
|---|---|
| Тег на грязном дереве / красных тестах | Шаг 2 |
| Release notes из голых commit messages | Перегруппировать и переписать для людей |
| Push и забыть | Шаг 5: деплой доведён до «живо» |
| Выкат без предупреждения (autoDeploy) | Сказать явно ДО push |
