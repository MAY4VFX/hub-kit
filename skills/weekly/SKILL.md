# Weekly — ритуалы недели

Единственный владелец W-лейблов (`W27` = ISO-неделя). Читает Hub Config
(`project_number`, `week_labels`) и HQ.md (список репо).

## /weekly --plan (начало недели)

1. **Текущая неделя**: `date +%V` → `W<NN>`. Создай лейбл `W<NN>` в затронутых репо.
2. **Разбор хвостов**: открытые issues с лейблом прошлой недели по всем репо HQ.md —
   таблицей: `issue | роль | статус на доске | тянуть в новую неделю?`. Пользователь
   решает: перенести (сменить W-лейбл) / в backlog (снять W) / закрыть.
3. **Набор недели**: пройди backlog + inbox — что берём? Выбранным: `W<NN>`, статус
   на доске, epic/dept-инварианты (правила `/route`).
4. **Итог**: план недели списком по ролям (`dept:*`), с дедлайнами в поле Deadline.

## /weekly --retro (конец недели)

1. **Факты**: закрытые за неделю issues по всем репо (`gh search issues --owner
   <owner> closed:>=<пн>`), work-records, коммиты. Сгруппируй по ролям.
2. **Ретро-issue**: в hub-репо, лейбл `retro:W<NN>` (создай): сделано / не сделано и
   почему / чему научились (кандидаты в `/learn`!) / фокус следующей недели.
3. **Гигиена**: незакрытые issues недели — предложи судьбу каждому (не закрывай сам);
   доска — унеси Done из активных колонок.
4. Добавь ретро-issue на доску; предложи сразу `/weekly --plan` следующей недели.

## Red flags

| Собираешься… | Вместо этого… |
|---|---|
| Закрыть неделю, молча закрыв недоделки | Судьбу каждой решает человек |
| W-лейбл будущего без плана | W-лейблы ставит только этот ритуал |
| Ретро без «чему научились» | Это главная секция — корм для /learn |
